# AMA Capability Commercialization System

Operating knowledge for how **AMA Solutions** commercializes the **Hanzo** sovereign operating system, the **Enso** intelligence-selection layer, and the wider **Lux** ecosystem. This repository is a **documentation foundation first** — a reviewable body of doctrine, positioning, claim governance, and decisions. Automation comes only after the documentation is approved.

> Canonical spellings: **Hanzo, Enso, Lux, AMA Solutions, Zach Kelling, MAIM.**

## What this repository is (and is not)

- **Is:** the commercialization doctrine, offer portfolio, ICP framework, customer-cohort model, claim-governance rules, evidence registry, escalation policy, validation plan, source manifest, and decision register.
- **Is not:** the delivery playbooks (those live in `AMA-Rabbit-Delivery-System`), product code, customer data, or credentials.

2. Direct Zach / Hanzo meeting notes and verified Hanzo documentation
3. Hanzo Technical Paper Compendium (company-published)
4. AMA / Claude portfolio pressure-test analysis
5. AMA strategic analysis
6. Research reports
7. Podcast transcripts (creative adversarial material — objections and language only, never verified evidence)

See `research/source-manifest.md` for the full mapping and per-source restrictions.

## How claims work here

Every material technical or commercial claim carries an **evidence status** — one of five canonical values: **Verified · Demonstrated · Founder-Reported · Roadmap · Unknown/Requires Verification** — kept separate from a **publisher/provenance** field (e.g. Hanzo-Published) and an **independent-verification** flag. Company-published material (including the Compendium) establishes what Hanzo *officially claims* — it does not independently verify performance. Nothing is marketed until it clears `docs/claim-governance.md`. See `docs/claim-governance.md` and `docs/evidence-registry.md`.
